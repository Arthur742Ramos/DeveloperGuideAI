# Documentation

Consolidated reference documents optimized for AI agent consumption. Each file is self-contained and can be read in a single pass.

## Files

| Document | Purpose | Size |
|----------|---------|------|
| [QUICK_REFERENCE.md](QUICK_REFERENCE.md) | One-page cheat sheet | ~150 lines |
| [ALL_PROMPTS.md](ALL_PROMPTS.md) | Every prompt template | ~500 lines |
| [GITHUB_COPILOT_COMPLETE.md](GITHUB_COPILOT_COMPLETE.md) | Full Copilot guide | ~350 lines |
| [CLAUDE_CODE_COMPLETE.md](CLAUDE_CODE_COMPLETE.md) | Full Claude Code guide | ~400 lines |
| [CODEX_CLI_COMPLETE.md](CODEX_CLI_COMPLETE.md) | Full Codex CLI guide | ~350 lines |

## For AI Agents

These documents are designed for efficient AI consumption:

- **Self-contained**: Each file has everything needed for its topic
- **No cross-references**: No need to read multiple files
- **Consistent structure**: Headers, code blocks, tables
- **Copy-paste ready**: Prompts can be used directly

## For Humans

Start with:
1. **QUICK_REFERENCE.md** for daily use
2. Tool-specific guide for your primary tool
3. **ALL_PROMPTS.md** when you need a specific template

## Relationship to Other Directories

| Directory | Purpose | When to Use |
|-----------|---------|-------------|
| `docs/` | Consolidated references | Reading/understanding |
| `prompts/` | Individual templates | Copying single prompts |
| `scripts/` | Executable wrappers | Running prompts |
| `instruction-files/` | Project templates | Setting up new projects |

The `docs/` files consolidate content from other directories into agent-friendly single files.
